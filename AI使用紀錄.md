# AI 使用紀錄

**楊子立（Yang Zi-Li）** ｜ 網頁設計期末作業 — 用 AI 共創一個完整的網站

---

## 一、我用了哪些 AI 工具、用來做什麼

這次主要用 **Claude（Anthropic）/ Claude Code** 當作我的協作夥伴，設計面和程式面都有用到：

- **設計面**：跟它一起發想整體視覺風格（深色科技感、配色、字體、進場動畫、數字雨背景、毛玻璃等），請它從設計師的角度給我方向和理由。
- **程式面**：請它依照我的需求寫出各區塊的 HTML / CSS / JavaScript，解釋程式怎麼運作，幫我找破版和動畫的問題並修正，也幫我把中文內容翻成英文。

---

## 二、關鍵對話截圖（設計發想 / 程式產出 / 除錯修改，各至少一例）

截圖依類別放在三個資料夾裡，檔名中 `-1`、`-2` 是 AI 的回應內容：

### 1.【設計發想】→ 資料夾「設計發想截圖」
- `9.png`：我請 AI 從資深 UI/UX 的角度，發想整個網站的視覺方向（核心概念、配色、字體、進場動畫、背景特效、毛玻璃、微互動、雙主題雙語）。
- `9-1.png`、`9-2.png`：AI 給的完整設計建議與理由。

### 2.【程式產出】→ 資料夾「程式產出截圖」
- `1.png`～`6.png`：我針對每個區塊下的需求指令（首頁、技能、專案、競賽榮譽、導覽列等）。
- `1-1 / 1-2 … 6-1 / 6-2.png`：AI 依需求實際寫出的 HTML / CSS / JS。

### 3.【除錯修改】→ 資料夾「除錯修改截圖」
- `7.png`、`8.png`：深淺色切換時，太陽和月亮圖示會同時跑出來。我請 AI 找原因，發現是 CSS 優先級（specificity）的問題——`.icon-btn svg` 的設定蓋過了 `.icon-sun` 的 `display:none`，改用更精準的選擇器後就修好了。

## 三、反思（約 190 字）

老實說，AI 幫我最多的是把想法直接變成能跑的程式。像數字雨、毛玻璃、進場解碼這些效果，我原本不知道怎麼寫，它幾乎馬上就給我能動的版本，省下很多試錯時間。但它不是每次都對：第一版數字雨太亮、把文字蓋掉，像素貓轉身時會被壓扁變形，翻譯和競賽名稱也和我的原意有出入，這些都得我自己抓出來。我的做法是用「讀者看不看得清楚」決定背景要壓多暗，用「動作合不合理」要它修掉貓咪變形，再照事實改掉翻譯。對我來說，AI 像個很快、但需要我盯著的助手，最後決定對不對、好不好的還是我自己。

---

## 四、我自己主導、動手改掉 AI 產出的地方

- 把太亮、蓋住文字的數字雨調暗，再加一層半透明遮罩確保可讀。
- 把原本平淡的淡出轉場，改成「文字模糊放大 → 黑幕上滑揭幕 → 內容浮現」更有層次的版本。
- 競賽榮譽從一開始的「卡片牆」改成「單欄列表 + 得獎標籤」，還把得獎標籤改成紅底白字，跟「參賽」明顯區隔。
- 把專業證照獨立成一區。
- 修掉深淺色圖示重疊、像素貓折返變形這些 bug。
- 逐一核對並修正全站的英文翻譯與競賽、系所名稱。
